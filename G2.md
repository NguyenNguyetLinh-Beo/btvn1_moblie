# G. Câu hỏi về bài làm
1. Tại sao dùng Nginx làm Reverse Proxy thay vì trỏ thẳng Tunnel vào Node-RED?

Tunnel cần một điểm vào phía server để xử lý request. Nginx đóng vai trò này: nhận request → kiểm tra → chuyển tiếp vào Node-RED.

Lợi ích:

- Làm entrypoint duy nhất
- Phân luồng request (/, /api)
- Tăng bảo mật (ẩn backend)
- Hỗ trợ mở rộng (load balancing)

Nếu trỏ thẳng vào Node-RED:

- Khó mở rộng
- Không tách frontend/backend
- Kém bảo mật
2. Sự khác nhau giữa Mount file và Mount thư mục trong Docker

Mount file

- Ví dụ: ./nginx/nginx.conf:/etc/nginx/nginx.conf
- Dùng cho config nhỏ
- Kiểm soát chính xác

Mount thư mục

- Ví dụ: ./myweb:/myweb
- Dùng cho code, data, logs
- Sửa ngoài host → container cập nhật ngay

3. Sửa index.html trên Ubuntu thì web có đổi không? Vì sao?

Có.
Vì nginx mount thư mục ./myweb → /myweb, nên khi sửa index.html trên host thì web cập nhật ngay.

5. Dùng chung network cho các services

Khai báo:
``
networks:
  mynetwork:
    driver: bridge
``
Gán vào service:
```
networks:
  - mynetwork
```
Lợi ích:

- Gọi nhau bằng tên container (http://nginx, http://myapi)
- Tách biệt với bên ngoài
- Dễ quản lý, bảo mật hơn

6. Tại sao phải đưa Cloudflare Token vào .env và không push lên GitHub?

Vì .env chứa thông tin nhạy cảm (token, config).
Nếu lộ:

- Có thể bị chiếm quyền tunnel
- Rò rỉ dữ liệu hoặc tấn công hệ thống

→ Phải thêm .env vào .gitignore.

7. Tại sao mount nginx.conf với :ro?

:ro = read-only

→ Chỉ cho đọc, không cho sửa

→ Tăng bảo mật, tránh bị thay đổi config

8. Dùng Cloudflare Tunnel có cần mở cổng không?

Không cần mở nhiều cổng.

Chỉ cần Nginx là entrypoint, sau đó Nginx proxy sang các service khác (ví dụ Node-RED).
